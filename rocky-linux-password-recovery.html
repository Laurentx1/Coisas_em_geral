OS: Rocky Linux 8.x (el8) Hypervisor: Proxmox VE Acesso: Console noVNC Tempo estimado: ~5 min
Parte 01 — GRUB
Passo 01
Reiniciar a VM pelo Proxmox
No painel do Proxmox, selecione a VM → clique em Shutdown → aguarde parar → clique em Start. Fique de olho no console (aba Console) para o próximo passo.
ℹ O menu do GRUB aparece por apenas alguns segundos. Deixe o console aberto antes de iniciar a VM.
Passo 02
Pressionar e no menu GRUB
Assim que aparecer o menu do GRUB com a entrada de boot do Rocky Linux, pressione e para entrar no modo de edição.
⚠ Se o GRUB passar rápido, use o campo Keyboard do Proxmox ou tente novamente. Janela de ~3 segundos.
Passo 03
Navegar até a linha linux
Use as teclas ↑ ↓ para chegar na linha que começa com linux ($root)/vmlinuz...
Passo 04
Ir ao final da linha com End
A linha linux é longa e vai quebrar visualmente. Pressione End para garantir que o cursor está no final real da linha — após biosdevname=0.
⚠ Este é o passo mais crítico. Se o cursor não estiver no final da linha linux, o parâmetro vai para a linha errada e não funciona.
Passo 05
Adicionar os parâmetros de recuperação
Com o cursor no final da linha linux, adicione um espaço e digite exatamente:
grub — kernel parameters
... biosdevname=0 rd.break enforcing=0
ℹ rd.break para o boot no initramfs. enforcing=0 desativa o SELinux temporariamente para evitar conflitos ao alterar a senha.
Passo 06 — Importante
Pressionar Ctrl + X para bootar
NÃO pressione Enter. O Enter criaria uma nova linha no GRUB e o parâmetro seria ignorado. Use diretamente Ctrl + X para iniciar com os parâmetros editados.
Parte 02 — Emergency Shell
ℹ O sistema vai bootar e exibir um prompt switch_root:/#. Você está no shell de emergência — execute os comandos abaixo na ordem.
Passo 07
Remontar o filesystem raiz com escrita
bash — emergency shell
# mount -o remount,rw /sysroot
Passo 08
Entrar no ambiente chroot
bash — emergency shell
# chroot /sysroot
O prompt muda para sh-4.4#. Agora você está operando dentro do sistema real.
Passo 09
Definir nova senha do root
bash — chroot
# passwd root
# # Digite a nova senha quando solicitado (2x) — sem eco na tela
ℹ A senha não aparece enquanto você digita. Digite normalmente e pressione Enter após cada confirmação.
Passo 10
Sair e reiniciar
bash — exit chroot e emergency shell
# exit # sai do chroot
# exit # sai do emergency shell → reinicia
A VM reinicia automaticamente e você consegue logar com a nova senha.
✓
Acesso Restaurado
Após o boot normal, faça login com root e a nova senha definida no Passo 09. A flag enforcing=0 é temporária — o SELinux volta ao modo normal no próximo boot.
